# Handbook Update Rules

只在需要判断是否更新说明书时再读这个文件.

## 输出位置

- 成品目录由 `context.md` 的 `handbook_output_dir` 指定
- 若未设置 `handbook_output_dir`, 由主控者指定输出位置, 不允许硬编码默认路径
- 模板永远放在 skill 的 `references/`

示例:
```yaml
handbook_output_dir: /mnt/d/obsidian-notes/15.项目说明书/
```

## 何时更新

只有命中下面任一条件时, 才创建或更新说明书:
- 新项目首次接入, 且这是长期项目
- 启动, 访问或验证方式变化
- 核心架构, 关键目录或主流程变化
- 出现人类协作必须知道的新坑, 新限制或新约定
- 当前说明书已经明显过期或会误导判断
- 某类细节已经稳定到值得抽成 reference

## 何时不更新

- 只是一次性实现细节
- 只是阶段进度变化
- 只是内部 memory 更新, 但不影响人类理解项目

## 更新方式

- 默认改主文档, 不轻易增文件
- 优先重写已有栏目, 不写流水账
- 主文档放高频信息, 低频细节再放 reference
